## 14. 日志可查

| # | 步骤 | 预期 | 通过 □ | 备注 |
|---|------|------|--------|------|
| 14.1 | 打开 `userData/logs/electron-*.log` | 含主进程、`backend-out`/`backend-err` 等行 | □ | |
| 14.2 | 后端标准输出 | 开发态子进程日志进入上述文件或控制台（视运行方式） | □ | |

**常见路径提示（Windows）**：Electron `userData` 多为 `%APPDATA%\排座助手`（与 `productName` 等相关），以 `app.getPath('userData')` 为准；其下 **`paizuo-data/paizuo.db`**（生产环境桌面 SQLite）、**`logs/electron-*.log`**。卸载默认**不删**用户数据，除非另行设计清理策略；详见 `docs/排座助手_桌面版交付说明.md` §2。

**关键词验收说明**：

- **`electron/preload.ts` 中的 `ipcRenderer.invoke("paizuo:getEnv")`**：Electron IPC，**不属于** Tauri `@tauri-apps/api` 残留。
- **`dist-electron/preload.js`**：由 `npm run electron:compile` 生成，已 `.gitignore`，**不作为**源码中 `invoke(` 残留判断依据。

## 16. 与本仓库静态配置对照（供验收填表参考）

| 项 | 位置 | 说明 |
|----|------|------|
| 前端 API 基址 | `src/api/client.ts` | 默认 `http://127.0.0.1:8765`，`VITE_API_BASE_URL` 可覆盖 |
| 开发后端脚本 | `package.json` → `dev:backend` | `uvicorn` 绑定 `127.0.0.1:8765` |
| Electron 端口 | `electron/paths.ts` | `PAIZUO_BACKEND_PORT` / `PAIZUO_PORT`，默认 8765 |
| 健康检查 | `electron/backendProcess.ts` | `GET /api/health` |
| 数据目录 | `electron/paths.ts` + `server/core/config.py` | `PAIZUO_DATA_DIR`；否则 Python 侧为 `server/data` |
| 打包后端 | `package.json` build.extraResources | `dist-backend/paizuo-backend.exe` |

**已修复（阶段 9）**：`PlansPage` 已从后端加载方案并支持新建 / 编辑 / 删除；统计卡片中「总方案数」为实时数量，其余仪表盘指标仍为占位（待接入活动/导出统计）。
