# 排座助手 · 桌面版交付说明

面向最终用户与实施同事的快速说明；技术细节以仓库 `README.md` 为准。

---

## 1. 用户安装说明

1. 获取安装包：`release/` 目录下的 **`排座助手-Setup-<version>.exe`**（NSIS）。
2. 双击运行安装向导；可按需修改安装目录（未启用「一键安装」）。
3. 安装完成后从开始菜单或桌面快捷方式 **「排座助手」** 启动。
4. 首次启动后，应用会在本机启动内嵌后端（`paizuo-backend.exe`）并打开主界面；若安全软件拦截，请将本应用加入信任列表。

*绿色免安装目录：使用 `build:desktop` 生成的 `release/win-unpacked/`，可直接复制分发（无安装向导）。*

---

## 2. 数据目录说明（固定路径约定）

以下路径均以 **Electron `app.getPath('userData')`** 为根，勿与安装目录混淆。

| 用途 | 相对路径 | 说明 |
|------|----------|------|
| 用户数据根 | （系统分配） | Windows 常见为 **`%APPDATA%\排座助手`**（与 `package.json` 的 `productName` 等相关，以运行时 `userData` 为准） |
| SQLite 数据目录 | **`paizuo-data/`** | 由主进程设置环境变量 **`PAIZUO_DATA_DIR={userData}/paizuo-data`**；库文件 **`paizuo.db`** 位于该目录 |
| 运行日志 | **`logs/`** | 主进程按日写入 **`electron-YYYY-MM-DD.log`**（见 `electron/logger.ts`）；后端子进程 stdout/stderr 也会汇入日志 |

**卸载与数据保留**：使用 NSIS 安装包卸载时，**默认会移除程序文件**，**通常不会删除**上述 `userData` 下的 `paizuo-data` 与 `logs`；用户主动重装后仍可能看到历史数据。**若需彻底清空数据**，请手动删除对应 `userData` 目录或其中子目录（务必确认备份，团队脚本勿对用户机批量删数据）。

**源码残留说明**：`electron/preload.ts` 中的 **`ipcRenderer.invoke("paizuo:getEnv")`** 属于 **Electron 主进程 ↔ 渲染进程 IPC**，与历史 **Tauri `invoke`** 无关；构建生成的 **`dist-electron/preload.js`** 已列入 `.gitignore`，不参与 Tauri 残留判断。

## 3. 开发环境说明

| 组件 | 命令 / 说明 |
|------|-------------|
| 前端 | `npm install` 后 `npm run dev`（端口 `1420`） |
| 后端 | `pip install -r server/requirements.txt` 后 `npm run dev:backend`（端口 `8765`） |
| API 代理 | Vite 开发服务器将 `/api` 代理到 `http://127.0.0.1:8765` |
| Electron 整栈 | `npm run electron:dev`（自动起 Vite + Electron + 开发态 Python） |

环境变量（可选）：

- **`VITE_API_BASE_URL`**：前端 API 基址（默认 `http://127.0.0.1:8765`）。
- **`PAIZUO_BACKEND_PORT` / `PAIZUO_PORT`**：后端监听端口（默认 `8765`，需与前端一致）。
- **`PYTHON`**：Electron 开发态拉起 uvicorn 时使用的 Python 可执行文件。

## 4. 构建命令说明（Windows）

| 命令 | 产物 |
|------|------|
| `npm run build:frontend` | `dist/`（静态前端） |
| `npm run build:backend` | `dist-backend/paizuo-backend.exe` |
| `npm run electron:compile` | `dist-electron/`（主进程/预加载脚本） |
| `npm run build:desktop` | `release/win-unpacked/`（未打包安装向导） |
| `npm run dist:win` | `release/排座助手-Setup-0.1.0.exe`（版本号随 `package.json`） |

**前置条件：**

1. 已将 **`build/icons/icon.ico`** 放入仓库（正式品牌包）；否则可能使用默认图标或构建报错。参见 `build/icons/README.md`。
2. 已成功执行 **`npm run build:backend`**，确保 `dist-backend/paizuo-backend.exe` 存在（`electron-builder` 的 `extraResources` 会打入安装包）。

若遇 **代码签名 / winCodeSign / 文件占用** 等问题，可先设置 `CSC_IDENTITY_AUTO_DISCOVERY=false`，关闭占用 `release/` 的进程后重试；必要时以管理员或「开发者模式」排除 symlink 限制（见团队内部分享）。

## 6. 常见问题（FAQ）

**Q：安装后双击无反应？**  
A：检查是否被安全软件拦截；查看 `%userData%` 下日志；尝试以 **`win-unpacked`** 目录直接运行 exe 观察控制台/日志。

**Q：接口连不上 / 一直加载？**  
A：确认后端进程已启动（任务管理器中有 `paizuo-backend.exe`）；防火墙是否拦截本机 `127.0.0.1` 端口；端口是否与默认 `8765` 一致。

**Q：数据存在哪里？**  
A：见上文「数据目录说明」；不要随意删除 `paizuo-data` 除非明确在做数据清理。

**Q：想恢复旧版 Tauri？**  
A：源代码在 **`archive/src-tauri/`**；需在仓库中恢复依赖与脚本，不在当前默认支持范围内。
